# Catch common misspellings in code and docs before review churn with codespell

Use codespell when an agent needs to scan a repository for common misspellings in source files, docs, comments, and config text, then optionally apply low-risk fixes. It is well suited to cleanup passes before review, documentation refreshes, and repo-wide polish tasks where a full grammar tool would be heavier than necessary. A user should invoke this instead of using their editor or a general writing tool normally when the job is repository-scale typo detection and remediation. The scope boundary is clear and skill-shaped: codespell only finds and fixes common misspellings in text-like project files, not broader prose editing, style coaching, or documentation platform management.
